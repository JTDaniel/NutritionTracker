# NutritionTracker

A full-stack personal nutrition and fitness tracking app with a web frontend, REST API backend, and React Native mobile app.

## Stack

| Layer | Technology |
|---|---|
| Web frontend | React + Vite + Tailwind CSS |
| Backend API | Node.js + Express |
| Database | MongoDB 7 (via Docker) |
| Mobile app | React Native + Expo (SDK 54) + NativeWind |
| Food data | USDA FoodData Central (~710k foods) |
| Containerization | Docker + docker-compose |

---

## Features

- **Food logging** — search 710k+ USDA foods, branded foods, and custom entries; log by date with serving size
- **Exercise logging** — search exercises, log duration and calories burned
- **Barcode scanner** — look up packaged foods by barcode (mobile)
- **Calorie & macro tracking** — daily targets, progress bars, net calories
- **Water tracker** — log intake with quick-add buttons
- **Weight tracker** — log weigh-ins, view trend chart
- **Body measurements** — track waist, hips, chest, neck, arms over time
- **Intermittent fasting timer** — live countdown, session history
- **Calorie history** — 30/60/90-day chart
- **Weekly summary** — bar chart of weekly averages
- **Streak tracker** — logging, calorie, water, exercise streaks with flame display
- **Gamification** — XP + leveling, 21 badge types, weekly challenges, streak shields, weight milestones
- **Diet support** — 12 diet types (keto, vegan, paleo, etc.) with compliance indicators
- **Custom foods** — create and manage your own food entries
- **Recipes** — build saved meals from food entries, log per serving
- **Dark mode** — full dark/light theme support on web and mobile
- **Profile persistence** — localStorage backup restores profile data after Docker volume wipe

## Running the Web App

### Prerequisites
- Docker + Docker Compose

### Start

```bash
docker compose up --build
```

- Web app: http://localhost:5173
- API: http://localhost:3002

The first run streams and seeds the USDA food database in the background (~5–10 min for full branded foods). Foundation + SR Legacy foods (~10k) are available immediately. Seed status is visible in the web UI.

### Stop

```bash
docker compose down
```

> **Note:** `docker compose down -v` deletes the MongoDB volume (all logged data). The app uses a localStorage backup to restore your profile on next launch, but food/exercise logs are not backed up this way.

## Running the Mobile App (Expo Go)

### Prerequisites
- Node.js 20 (via nvm — see below)
- Expo Go installed on your Android/iOS device
- [Tailscale](https://tailscale.com) on both your computer and phone (for remote access)

### Install Node 20 via nvm

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20
nvm alias default 20
```

### Install and run

```bash
cd mobile
npm install --legacy-peer-deps
npx expo start
```

Scan the QR code with Expo Go. On first launch, open **More → Settings** and enter your backend URL:

```
http://<tailscale-ip>:3002
```

Get your Tailscale IP with `tailscale ip -4`. Tap **Test Connection** to verify.

### Run over Tailscale (off local network)

```bash
tailscale ip -4        # get your IP
npx expo start --host <tailscale-ip>
```

## Building a Standalone Android APK

Requires Java 17 (via SDKMAN) and Android command-line tools.

### One-time setup

```bash
# SDKMAN + Java 17
curl -s "https://get.sdkman.io" | bash
source ~/.sdkman/bin/sdkman-init.sh
sdk install java 17.0.11-tem

# Android command-line tools
mkdir -p ~/Android/sdk/cmdline-tools
cd ~/Android/sdk/cmdline-tools
wget https://dl.google.com/android/repository/commandlinetools-linux-11076708_latest.zip
unzip commandlinetools-linux-11076708_latest.zip
mv cmdline-tools latest

# Environment variables (add to ~/.bashrc)
export ANDROID_HOME=~/Android/sdk
export JAVA_HOME=~/.sdkman/candidates/java/current
export PATH=$PATH:$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools

# Accept licenses and install SDK components
yes | sdkmanager --licenses
sdkmanager "platform-tools" "build-tools;35.0.0" "platforms;android-35"
```

### Build APK

```bash
cd mobile/android
./gradlew assembleRelease
```

Output: `mobile/android/app/build/outputs/apk/release/app-release.apk`

Serve it to your phone:

```bash
cd mobile/android/app/build/outputs/apk/release
python3 -m http.server 8000
# open http://<your-ip>:8000/app-release.apk on phone
```

Enable **Install from unknown sources** in Android settings before installing.

## API Reference

All endpoints are prefixed with `/api`.

| Method | Path | Description |
|---|---|---|

| GET | `/logs/exercise?userId=&date=` | Get exercise log |
| POST | `/logs/exercise` | Add exercise to log |
| DELETE | `/logs/exercise/:logId/item/:index` | Remove exercise item |
| GET | `/weight?userId=` | Get weight entries |

| GET | `/fasting/active?userId=` | Get active fast |

## Gamification

**XP actions:**

| Action | XP | Limit |
|---|---|---|
| `FOOD_LOG` | +10 | Once per day |
| `CALORIE_TARGET_HIT` | +25 | Once per day |
| `EXERCISE_LOG` | +15 | 3× per day |
| `WEIGH_IN` | +10 | Once per day |
| `WATER_GOAL_HIT` | +10 | Once per day |
| `FASTING_COMPLETE` | +20 | Unlimited |

**Level formula:** `level = floor(1 + sqrt(xp / 100))`

Streak shields are awarded every 5 levels. A shield protects one day's streak from being broken.

## Diet Types

Keto, Low-Carb, Vegan, Vegetarian, Pescatarian, Mediterranean, Paleo, DASH, High-Protein, Whole30, Gluten-Free, None.

Each diet shows compliance indicators on food cards based on macro thresholds (keto, low-carb, DASH) or ingredient keyword matching (vegan, paleo, gluten-free, etc.).
